Post 3 - On Arrays and Hashes

So today's lesson deals with how we store and recall information in Ruby. One way to do it is in an ordered list called an array. Another way to do it is with a series of key/value pairs that act like a dictionary. Let's take a look at some differences and similarities...

Arrays

Arrays are just ordered lists like 1, 2, 3 or biscuits, gravy, fried_okra. Or even 1, gravy, 3.

When you write them in Ruby, they look like this...

sweet_array = [0, 1, 2]

So this is just a very simple example that sets sweet_array equal to an ordered list with the index value as the integer in that slot.

To call the variable at index 2, you would say...

sweet_array = [2]
				=> 2

Arrays are most useful when need to select an onject from a consistent, consecutive index.

Hashes

Hashes are basically lists of key/value pairs where the keys point to the value. So like an array can say biscuit, but a hash can say breakfast is a biscuit.

While arrays use [], hashes use {}, like so...

sweet_hash = {"key 1" => "value 1",
			 "key 2" => "value 2",
			 "key 3" +. "value 3"
			 }

You can recall a certain value using

first_value = sweet_hash["key 1"]
==>"value 1"

The Takeaway

So there are at least 2 different ways to store and retrieve information, and both are explained in most simple terms above.
