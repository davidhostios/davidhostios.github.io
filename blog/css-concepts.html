Post 2 - On Positioning

Static (Normal)
Relative
Absolute
Fixed

These four properties control the way you can set the initial position of your HTML elements when you are building a web page.

To get anything done, you have to understand that these are moved by manipulating the offset properties top, bottom, left, and right, so the command would look like

p {

position: relative;

top: 40px;

}

Just as a for instance.

Below is a breakdown of the difference between each of them...

Static

So static positioning just says that this element will just stack below the element above it, like the paragraphs you are reading now. Just the normal flow of things. It really is that simple.

You don't actually have to set position: static to use this as it is the default setting.

Relative

Relative positioning moves an HTML element relative to where it would have been in the static (normal) flow of things. So for instance, this paragraph should be moved 55px to the right and down 35px by adding

p {

position:relative;

top: 35px;

left: 55px;

}

Absolute

Absolute positioning removes an element from the normal flow of a web page and uses offset properties to position itself within the content, like the words

"Aw Yea, Absolute, Yall"

hidden in plain sight somewhere on this page - can you find it?

Aw Yea, Absolute, Yall

by adding this code

p {

position: absolute;

top: 305px;

left: 55px;

}

Fixed

Fixed position will remove your element from normal flow, but unlike absolute, it will keep you content where you tell it to stay in the browser instead of in the content. You'll never guess where my example of the below code is...

Aw Yea, Fixed, Yall

p {

position: fixed;

top: 0;

right: 0;

}
