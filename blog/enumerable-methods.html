Post 4 - On Enumerable Method Map

To understand Enumerable method map, you first have to get that Enumerable is a class that has a bunch of built in collection related behaviors.Another thing to know is that these methods often pair with each.

The map method is one of the most powerful collection operations you can use in Ruby. Map will always result in an array. So, for instance, if you had a list and you wanted to do some formatting (which would have been useful to the author this week...), you could use map like this... 

> grocery_list = %w{ diapers wipes formula beer }
=> ["diapers", "wipes", "formula", "beer"]
> grocery_list.map {|grocery_list| grocery_list.upcase}
=> ["DIAPERS", "WIPES", "FORMULA", "BEER"]

As you can see, after getting diapers, wipes, and formula, sometimes it is necessary to get beer.

Seriously though, the new array is the same size as the original but we used map to create a new object. In this way, it differs from .each.

You gotta be careful when you use .map because if your code block that you execute contains a puts (kernel method that puts a string on a new line) it will evaluate to nil. Which isn't terrible, but not likely what we are trying to do.

There also exist the map! version of this method, but it may not be logically useful for much more than formatting an existing array (as opposed to like a range of numbers or something).

In summation, the Enumerable map method is useful for effecting a collection of information and creating a new object. When I say it like that, it doesn't sound like much, but foreal it's pretty handy.
